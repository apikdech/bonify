# Receipt Manager — Phase 1: Core Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task.

**Goal:** Build the complete backend foundation, LLM vision pipeline with Temporal workflows, Telegram/Discord bots, and PWA dashboard with all core features.

**Architecture:** Go backend with chi router, PostgreSQL for persistence, Redis for sessions/cache, MinIO for object storage, Temporal for async workflows, SvelteKit PWA frontend with static adapter served by Go.

**Tech Stack:** Go 1.25+, chi, pgx, Temporal, any-llm-go, SvelteKit, Tailwind CSS, Chart.js, MinIO, PostgreSQL 16, Redis 7

**Duration:** ~6 weeks

---

## File Structure

**Backend:**
```
backend/
  cmd/api/main.go          # API server
  cmd/worker/main.go       # Temporal worker
  internal/
    config/config.go       # Environment config
    db/db.go + migrations/ # Database
    middleware/auth.go     # JWT, rate limiting
    handler/*.go          # HTTP handlers
    service/*.go          # Business logic
    repository/*.go       # Data access
    model/*.go            # Domain models
    workflow/*.go         # Temporal workflows
    bot/telegram.go       # Telegram bot
    bot/discord.go        # Discord bot
```

**Frontend:**
```
frontend/
  src/lib/api.ts         # API client
  src/lib/auth.ts        # Auth store
  src/routes/            # SvelteKit pages
  src/components/        # Reusable components
```
